Intro

Nitobi
  - services company
  - build on open source products
  - mobile centric
  - we write a lot of ruby (other stuff too)
  
What brought us here (ruby, toronto, epic conference)

We are really here to talk about mobile development

The ruby mindset
  - remove ceremony
  - don't get carried away with name spacing
  - put your effort where it counts
  - programmer happiness
  - efficiency
  
Love ruby design but not the environment (people are solving this problem)

Problems We have
  - deployment is a bitch
  - consumes a lot of resources
  - not on devices by default

Basically ruby is not the best choice for mobile ...today

Internet is here!

HTML / CSS / JAVASCRIPT

Javascript is small, fast, and FUCKING EVERYWHERE!

Apple dashcode && palm SDK (this is the right approach. rob and brian will elaborate on this)

Javascript solves most of our mobile problems

Smart Platform
  - server-side javascript
  - code sits in git repo (consumes no resources)
  - fires up a new VM for each request
  - scales infinitely (in theory)
  - Sammy framework (sinatra equivalent in js)
  - available now!
  
Smart deserves and entire talk but let me show you hello world with sammy
(2 min)

WTF? I thought you were here to talk about phonegap?

What is PhoneGap?
  Cross platform mobile development build on open web standards
  - Javascript API that exposes Core functionality of mobile devices
  - based on HTML5 spec
